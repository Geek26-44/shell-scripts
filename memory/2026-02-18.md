# 2026-02-18

## Безопасность — критический инцидент 🔴

### Проблема
- Порт 5900 (VNC/Screen Sharing) был проброшен на роутере в интернет
- 59+ ботнетов активно брутфорсили VNC (91.92.243.x, 178.16.x.x, 158.94.209.x)
- CLOSE_WAIT соединения забивали лимит — невозможно подключиться из локалки

### Решение
- Удалён проброс VNC-MacMini (TCP:5900) на роутере AT&T BGW320
- Оставлен только WireGuard-VPN (UDP:51820)
- Перезапущен Screen Sharing (sudo killall screensharingd + переключение в Settings)
- VNC теперь доступен только из LAN (192.168.1.0/24) и через WireGuard (10.66.66.0/24)

### Security Monitor — построен ✅
- `geek-dashboard/backend/security_monitor.py` — standalone скрипт мониторинга
- Таблицы: `security_alerts`, `security_baselines`
- 8 API эндпоинтов: alerts CRUD, stats, status, scan
- Вкладка "🛡️ Безопасность" в дашборде
- Cron job: каждые 15 минут, алертит если critical/high
