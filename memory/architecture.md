```

│                      Дима (CEO)                          │
│                         ↓                                │
│              Ввод: текст / голос / файл                   │
│              + опционально: /model opus | codex | gemini │
│                         ↓                                │

│  │         Qwen 14B — ДИСПЕТЧЕР (локально)      │        │
│  │                                              │        │
│  │  • Видит ВСЁ: SOUL, USER, MEMORY, историю   │        │
│  │  • Классифицирует задачу                     │        │
│  │  • Фильтрует PII перед отправкой наружу      │        │
│  │  • Переводит RU→EN для внешних моделей       │        │
│  │  • Адаптирует ответ обратно (стиль, RU)      │        │
│  │                                              │        │
│  │  Если /model указан → пропуск классификации, │        │
│  │  прямая отправка на выбранную модель          │        │
│  │  (PII фильтр всё равно работает!)            │        │
│  └──────────┬───────────────────────────────────┘        │
│             ↓                                            │
│  ┌──────────────────────────────────────────────┐        │
│  │          МАРШРУТИЗАЦИЯ                        │        │
│  │                                              │        │
│  │  AUTO (по умолчанию):                        │        │
│  │  ├── SIMPLE → Qwen сам (локально)            │        │
│  │  ├── COMPLEX → внешняя модель (EN, без PII)  │        │
│  │  └── SENSITIVE → см. правила ниже            │        │
│  │                                              │        │
│  │  MANUAL (/model <name>):                     │        │
│  │  ├── /model qwen → принудительно локально    │        │
│  │  ├── /model opus → Claude Opus               │        │
│  │  ├── /model codex → Codex CLI (OpenAI)       │        │
│  │  ├── /model gemini → Gemini CLI (Google)     │        │
│  │  └── /model deepseek → DeepSeek (локально)   │        │
│  │                                              │        │
│  │  ⚠️ При любом MANUAL выборе внешней модели:  │        │
│  │  PII фильтр ВСЕГДА активен, не отключается   │        │
│  └──────────────────────────────────────────────┘        │
└──────────────────────────────────────────────────────────┘
```

## 3. Приватность и фильтрация

### 3.1 Разделение контекста

| Данные | Qwen (локально) | Внешние модели |
|--------|-----------------|----------------|
| SOUL.md (персона) | ✅ | ❌ |
| USER.md (Дима) | ✅ | ❌ |
| MEMORY.md | ✅ | ❌ |
| История чата | ✅ полная | Только релевантный кусок |
| Задача | ✅ | ✅ на EN, обезличенная |
| Код и файлы | ✅ | ✅ полностью (иначе галлюцинации) |
| Технический контекст | ✅ | ✅ если релевантно |

**Принцип: убираем WHO, оставляем WHAT.**

### 3.2 Три режима задач

**SIMPLE** → Qwen отвечает сам, ничего не уходит наружу.

**COMPLEX** → Внешняя модель получает:
- ✅ Полное описание задачи (EN)
- ✅ Код, конфиги, логи — как есть, полностью
- ✅ Технический контекст (железо, ОС, версии)
- ❌ PII (имена → "the user", города → убрать, даты рождения → убрать)
- ❌ Персона, стиль общения

**SENSITIVE** → Qwen пробует сам. Если не справляется — анонимизирует:
- Финансы: суммы ✅, реквизиты ❌, контрагенты ❌, откуда/куда ❌
- Идеи: абстрактные параметры (категория, TAM, цена, тип) ✅, сама идея ❌
- Личное: ситуация в общих чертах ✅, идентификация ❌

### 3.3 Абстрагирование идей

```
Дима: "Оцени рынок для [моя идея]"
  ↓
Qwen (понимает идею полностью, локально):
  → Внешней модели: "Evaluate SaaS in [broad category],
    targeting [segment], ~$X/mo, TAM ~N users.
    Key risks and failure modes?"
  ↓
Opus: анализирует паттерн, не зная конкретику
  ↓
Qwen: накладывает анализ на реальную идею + адаптирует
  ↓
Дима: получает полный ответ
```

### 3.4 Self-check

После абстрагирования Qwen проверяет: "Можно ли по этому промпту восстановить оригинальную идею?" Если да → переписывает.

### 3.5 Ручной выбор модели + приватность

**ВАЖНО:** Даже при `/model opus` (принудительный выбор) — PII фильтр не отключается.
Единственный способ отправить всё как есть — `/model qwen` (локально).

```
/model opus "проанализируй мой бизнес-план"
  ↓
Qwen: пропускает классификацию, НО:
  1. Фильтрует PII
  2. Переводит на EN
  3. Отправляет на Opus
  4. Адаптирует ответ
```

## 4. Multi-pass обработка

Приоритеты: **приватность + качество > скорость**. Latency 40-60 сек допустим.

```
Pass 1: Классификация задачи + план действий     (~3 сек)
Pass 2: Фильтрация PII + абстрагирование         (~5 сек)
Pass 3: Перевод RU→EN + формирование промпта      (~5 сек)
Pass 4: Self-check (можно ли восстановить PII?)   (~3 сек)
   → Внешняя модель                               (~10-30 сек)
Pass 5: Адаптация ответа + стиль + RU             (~5 сек)
                                          Итого:   ~30-50 сек
```

При ручном выборе модели Pass 1 пропускается.

## 6. Инфраструктура

### Железо
- Mac mini M4, 16GB RAM, 256GB SSD
- + 2TB внешний диск (dock-станция) — модели, документы, БД

### Софт
- **Ollama** — локальные модели (Qwen, Gemma, DeepSeek)
- **OpenClaw** — оркестрация, сессии, cron
- **Codex CLI** — OpenAI для кода
- **Gemini CLI** — Google для ресёрча
- **ChromaDB** (или аналог) — локальный RAG, векторный поиск
- **Dashboard** — Kanban, статусы отделов, проекты

### Ограничения 16GB RAM
- Одна модель Ollama активна одновременно (~9GB)
- Агенты работают последовательно, не параллельно
- Это конвейер, не параллельный мульти-агент
- При переключении моделей — задержка на загрузку (~5-10 сек)

## 7. Проверка работоспособности

### ✅ Что точно работает на текущем железе
- Qwen 14B как роутер + простые задачи — проверено
- Внешние CLI (Codex, Gemini) — отдельные процессы, не грузят RAM
- PII фильтрация — regex + Qwen (лёгкая задача)
- Перевод RU→EN — Qwen справится
- Последовательная обработка отделов — работает

### ⚠️ Потенциальные узкие места
- **Качество абстрагирования идей** — Qwen 14B может неточно обобщить сложную идею. Решение: self-check + итеративная доработка
- **Контекст между вызовами** — каждый вызов внешней модели изолирован. Решение: Qwen передаёт необходимый контекст из истории
- **Переключение моделей Ollama** — 5-10 сек на загрузку при смене. Решение: кешировать активную модель, менять реже
- **256GB SSD** — заполнится быстро с несколькими моделями. Решение: 2TB диск (запланирован)

### ❌ Что НЕ будет работать
- Параллельная работа нескольких Ollama моделей (16GB не хватит)
- Большие модели (70B+) — физически не влезут
- RAG на больших объёмах без внешнего диска

## 9. Принципы (неизменные)

1. **Приватность** — ничего наружу без фильтрации. PII фильтр всегда активен.
2. **Качество > скорость** — лучше 50 сек хорошего ответа, чем 10 сек галлюцинации.
3. **WHO убираем, WHAT оставляем** — внешние модели не знают кто спрашивает.
4. **Код и файлы полностью** — не резюмировать, иначе галлюцинации.
5. **Ручной override** — всегда можно принудительно выбрать модель.
6. **Фильтр не отключается** — даже при ручном выборе внешней модели.
7. **Каждый инструмент в своей нише** — Qwen роутит, не исполняет сложное.
8. **Юридика ≠ юрист** — LLM для драфтов и анализа, финальные решения за профессионалом.
