## Архитектура

### Hub-and-Spoke (Центральный узел)
- **Я (Geek)** — центральный координатор
- **Sub-агенты** — изолированные исполнители
- **Коммуникация** — агенты НЕ общаются друг с другом, только через меня
- Каждый агент имеет свою структуру и контекст

## Стратегия LLM

### Текущее состояние: Гибридная модель
- **Облачные провайдеры:** OpenAI, Gemini, Claude
- Требуют интернет-соединения
- Используем для сложных задач

### Развитие: Локальные LLM
- Параллельно развиваем собственные локальные модели
- Независимость от внешних провайдеров
- **Fine-tuning:** позже, после сбора информации и документации

### Железо
- **Платформа:** Apple-only (Apple Silicon)
- **Текущее:** Mac mini M4, 16GB RAM, 256GB SSD
- **Ограничения:** 7-13B модели комфортно, 30B впритык, 70B не влезет
- **Будущее:** +2-4TB внешний диск для моделей

## Правила

### 🔒 ГЛАВНОЕ: Приватность и изоляция

**Ничего не уходит наружу без явного разрешения.**

- **Документы** — обрабатываем только локально
- **Телеметрия** — отключаем везде где возможно
- **Софт** — проверяем что не сливает данные разработчикам
- **Сеть** — мониторим исходящие соединения
- **LLM** — причина перехода на локальные модели

**Checklist для нового софта:**
- [ ] Есть ли телеметрия? Можно отключить?
- [ ] Какие данные собирает?
- [ ] Open source? Можно проверить код?
- [ ] Работает офлайн?
