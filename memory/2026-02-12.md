# 2026-02-12

## Архитектурная сессия — полная ревизия мульти-агентной системы

### Ключевые решения:
- **Перевод RU→EN** перед отправкой на внешние модели — экономия 40-60% токенов
- **Разделение контекста**: убираем WHO (PII), оставляем WHAT (задачу, код, тех.контекст)
- **Приоритеты**: приватность + качество > скорость. Latency 40-60 сек допустим.
- **Multi-pass** подход для Qwen: классификация → PII фильтр → перевод → self-check → внешняя модель → адаптация
- **Self-check**: после абстрагирования Qwen проверяет "можно ли восстановить идею по промпту?"
- **Ручной выбор модели** (`/model opus|codex|gemini|qwen|deepseek`) — PII фильтр всегда активен даже при ручном выборе

### Три режима задач:
- SIMPLE → Qwen сам
- COMPLEX → внешняя модель (полный тех.контекст, без PII, на EN)
- SENSITIVE → Qwen сначала сам; если не может — анонимизирует:
  - Финансы: суммы ✅, реквизиты ❌, контрагенты ❌
  - Идеи: абстрактные параметры ✅, сама идея ❌ (Qwen → абстрактные метрики → Opus оценивает паттерн → Qwen накладывает на реальную идею)
  - Личное: ситуация ✅, идентификация ❌

### Виртуальная компания — отделы:
